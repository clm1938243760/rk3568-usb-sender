## 最新更新

### 本次更新内容
- 修复 RK3568 开发板 USB U盘自动上传逻辑
- 优化 `usb_sender.sh` 的镜像检测与处理流程
- 调整 USB 重新挂载方式，改为调用系统原生 `S50usbdevice restart`
- 提高不同电脑下的 U盘识别稳定性
- 修复发送文件后 USB 重连异常问题
- 增加对镜像变化、新文件检测、上传结果的日志输出

### 当前实现功能
- 开发板通过 OTG 口模拟 U 盘
- Windows 可将文件写入虚拟 U盘镜像
- 开发板检测镜像变化后自动扫描新文件
- 新文件可自动上传到服务器
- 上传完成后自动恢复 U盘连接状态

### 当前项目文件说明
- `usb_sender.sh`：主发送脚本，负责检测镜像变化、扫描新文件并调用上传
- `http_upload_file.py`：HTTP 上传脚本，负责将文件发送到服务器
- `S99usb_sender`：开机自启脚本
- `README.md`：项目说明文档

### 已知说明
- 当前方案中，USB U盘与板子本地处理共用同一个镜像文件
- 在上传处理阶段，U盘可能会出现短暂断开并重新连接，这是当前机制下的正常现象
- OTG 热插拔识别稳定性依赖当前设备树与系统 USB 配置
